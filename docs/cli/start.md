Start Command

The "start" command is used to run a Najumi.js application in production mode.

After building the application using the "build" command, the "start" command launches the optimized server that runs the compiled production version of the application.

This server is designed for stability, performance and scalability.

---

Basic Usage

To start the production server, run:

najumi start

This command runs the compiled application generated by the build process.

The server will start and begin listening for incoming requests.

Example output:

🚀 Najumi Production Server Started
🌐 http://localhost:3000

Once the server starts, the application will be available in your browser.

---

Prerequisites

Before running the "start" command, the application must first be built.

Run:

najumi build

This generates the production-ready files in the "dist" directory.

After the build is complete, the production server can be started.

---

Production Workflow

A typical production workflow looks like this:

1. Build the application:

najumi build

2. Start the production server:

najumi start

3. The application runs using optimized production code.

---

Production Server

The production server is designed to provide a stable runtime environment.

It is optimized for:

- efficient request handling
- reduced memory overhead
- stable long-running processes

Unlike the development server, the production server does not include development features such as hot reload.

This ensures that the server runs with minimal overhead.

---

Default Server Port

By default, the production server runs on port:

3000

You can access the running application at:

http://localhost:3000

Server configuration can also be modified through the project configuration file.

---

Running in Different Environments

Applications can be deployed to various environments such as:

- cloud servers
- container platforms
- virtual machines
- traditional hosting environments

The production server generated by Najumi.js is designed to run reliably in these environments.

---

Stopping the Server

To stop the running production server, press:

Ctrl + C

This will terminate the server process.

---

Summary

The "start" command launches the production server for a Najumi.js application.

It runs the optimized application generated by the build process and prepares the system to handle production traffic.
