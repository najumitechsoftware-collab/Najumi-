Introduction

Najumi.js is a modern full-stack development framework designed to help developers build scalable, secure and high-performance applications using a unified architecture.

Unlike traditional frameworks that only focus on either frontend or backend, Najumi.js provides a complete ecosystem for building modern software systems.

With Najumi.js, developers can build:

- web applications
- APIs
- scalable backend systems
- interactive frontend interfaces
- data-driven applications

All using a single integrated framework.

---

Why Najumi.js Exists

Modern application development has become increasingly complex.

Developers often need to combine many different tools and libraries just to build a single application. A typical modern stack might include:

- a frontend framework
- a backend framework
- a database ORM
- a queue system
- a storage service
- a caching layer
- an authentication system
- secret management

Managing all these systems separately can introduce complexity, security risks and performance overhead.

Najumi.js was created to simplify this process by providing a unified ecosystem where these systems are designed to work together from the beginning.

---

Core Philosophy

Najumi.js is built around three fundamental principles.

Security First

Security is integrated into the framework architecture itself.

Instead of relying on external libraries for critical protections, Najumi.js includes built-in security systems such as:

- request validation
- secure middleware pipelines
- secret management
- authentication strategies
- storage protection layers

This approach helps developers build applications that are secure by default.

---

Ultra-Fast Performance

Najumi.js is designed for high performance.

The framework includes optimized internal systems such as:

- an ultra-fast routing engine
- an optimized rendering system
- an efficient caching layer
- scalable background job processing

These systems are designed to handle high workloads while maintaining low latency.

---

Developer Simplicity

Najumi.js aims to make development simple and intuitive.

Developers should be able to focus on building features rather than managing infrastructure.

This is achieved through:

- a clear project structure
- an integrated CLI
- modular architecture
- predictable framework conventions

---

The Najumi Ecosystem

Najumi.js includes several internal systems designed to work together as a unified development platform.

Najumi View

Najumi View is the frontend UI framework.

It provides:

- component system
- reactive state management
- fast rendering engine
- layout system
- client hydration

This allows developers to build interactive user interfaces efficiently.

---

Najumi Atlas

Atlas is the database system used by Najumi.js.

It provides an abstraction layer for interacting with databases while maintaining strong performance and scalability.

Atlas supports multiple database connectors including:

- PostgreSQL
- MySQL
- MongoDB

---

Najumi Harbor

Harbor is the storage engine used for managing files and uploads.

It provides a secure and scalable file storage system capable of handling large uploads and high volumes of stored files.

---

Najumi Pulse

Pulse is the caching system used by Najumi.js.

It provides high-performance caching capabilities designed to reduce database load and improve application speed.

---

Najumi Orbit

Orbit is the background job and queue system.

It allows developers to run asynchronous tasks such as:

- sending emails
- processing uploads
- running scheduled jobs
- background data processing

---

Najumi Vault

Vault is the secret management system used by Najumi.js.

It is responsible for securely storing sensitive data such as:

- API keys
- database credentials
- authentication secrets
- environment variables

---

Modular Architecture

Najumi.js follows a modular architecture.

Applications can be divided into independent modules, each responsible for a specific part of the system.

Modules may include:

- APIs
- services
- models
- background jobs
- event handlers

This approach helps large applications remain organized and maintainable.

Full-Stack Development

Najumi.js enables full-stack development using a single framework.

Developers can build:

- frontend interfaces
- backend APIs
- database models
- background jobs
- storage services

All within one consistent architecture.
